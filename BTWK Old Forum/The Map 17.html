  1. DC123456789 - 09-11-2015, 19:27
    As for adding provinces, this page is still pretty good for the basics, even if kind of outdated. A few notes:
    - Don't use Excel for .csv files, use Notepad++. It's much easier, and you're less likely to accidentally screw up.
    - The province name actually matters now, it has to be the same name in the definition.csv and in the name of the province history (but you should have it the same anyway).
    - Don't bother grouping all the sea zones together in one block, we're way past that point. The way it's setup in default.map is pretty intuitive and you should be able to figure out how it works fairly quickly - if you don't, just ask. So you can ignore everything that refers to that.
    - As mentioned, using the Nudge tool is a far easier way of doing positions. It's not that important though, and can easily be added later. There are couple of things about the Nudge tool that you need to know before you can use it properly though.
    - You can just make the provinces under the nearest existing de jure duchy and copy nearby provinces' history titles for now, and we can sort it out later.
    - For the flags, you can just use the rebel flag as a placeholder (it's k_rebels).
    - The province name localisations are in Province Names.csv, and the settlement name localisations are in Titles - Settlements.csv.
    - Adjectives aren't required, don't bother if you don't want to add them.

    Hopefully it all makes sense. If it doesn't or you get stuck/crashes, just ask.

  2. DC123456789 - 10-11-2015, 07:47
    Also, don't forget to add your new province ids in the appropriate places in map/climate.txt (defines what seasons affect the province), map/continent.txt (defines the "continent", i.e. contiguous land zone (including passable straits) - required for AI pathfinding), and map/island_region (which is almost the same as continent.txt).
