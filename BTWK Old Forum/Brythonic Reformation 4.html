  2. DC123456789 - 22-03-2015, 11:18
    Yeah, that seems fine. The quest should end up consuming a lot of your life.

    So our system for the Archdruid succession is going to be that it rotates between landed druids/archdruids once the previous one dies, correct? This is what I'm thinking of:
    - The next Archdruid is always decided in advance, getting some sort of "Archdruid Successor" (Honorary/Religious) title. When the Archdruid dies or changes religion, a hidden event triggers that gives the Archdruid title to the successor and starts the event chain for choosing the next one.
    - All landed druids and archdruids get an event asking them if they want to be a candidate to be the next Archdruid.
    - If they accept, their "score" is calculated through various things, such as piety, prestige, traits, rank, age, learning, etc. Probably piety, learning, and age should matter the most. Perhaps there should also be a random factor to the score to make things a little more interesting.
    - Once all the scores have been calculated, the character with the highest score will be declared the successor.

    Of course, this would require there to be more landed druids than there currently are, otherwise there isn't really a contest. Perhaps taking the theology focus would give you events to slowly work your way up to the requirements of being a druid? There could also be hidden events to make barony-level temple holders druids more often.
    Edit
  3. Ols - 22-03-2015, 11:29
    Yes, we need more Druids. Taliesin talks about how one becomes a Druid in Excalibur. Perhaps we could revamp the theology focus so that it culminates in becoming a Druid if a Celtic Pagan takes it. I'm sure a number of the events leading to Theologian could simply be relocalised so that they instead refer to Druidic training. Certainly we should have maintenance events that assign the Druid trait to temple holders fairly regularly. I believe I've set it so Celtic Pagan religions can hold temples, so we'd need to be careful that it doesn't make loads of feudal rulers into Druids indiscriminately. There's a musician trait somewhere in our files that could well be changed to "Bard" and made an alternative result of training to be a Druid but refusing to take the priestly staff. In any case, I believe it'd be a case of relocalising stuff and connecting the dots with old assets. I'm happy to write long sets of localisation and posting them here if we want a particularly descriptive event chain. I wont have SVN for the next month as I'll be away campaigning/working, so that's probably the best I can do.

    I believe the succession process you're suggesting would work well, but it ultimately boils down to whatever you can make work.
